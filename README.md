# EquipoA_SistemaPosgrados

Sistema de Gestión para Funciones Administrativas y Académicas de un Posgrado

## Tecnologías:

HTML
CSS
JavaScript
React
Python
MongoDB

## Guía de instalación frontend

Sigue estos pasos para probar el proyecto localmente:

1. Clonar el repositorio

```bash
  git clone "url-del-repo-sin-comillas"
```

2. Actualizar el repositorio

```bash
   git pull origin "aqui-va-la-rama"
```

3. Entrar a la carpet del frontend

```bash
   cd frontend
```

4. Instalar dependencias con NPM

```bash
    npm install
```

5. Abrir el servidor local

```bash
    npm run dev
```

## Reglas de Contribución

Para garantizar la estabilidad del proyecto, es obligatorio cumplir con el siguiente protocolo de trabajo:

### Commits Restringidos

- **PROHIBIDO:** Realizar commits directos a las ramas `main` o `develop`.
- **Sanción:** Cualquier commit realizado en estas ramas sin aprobación previa será **deshecho (reverted)** de inmediato.

### Flujo de Ramas y Merges

- **Ramas Individuales:** Cada integrante debe trabajar en su propia **rama general** y crear **subramas** específicas para cada tarea o funcionalidad.
- **Merge a `develop`:** Antes de integrar cambios, debe a ver comunicacion hacia los lideres tecnicos no necesariamente los lideres de equipo, los roles estan por ser definidos
  y arquitecto de software, de los cambios y obtener la aprobación correspondiente.
- **Merge a `main`:** Este es un proceso crítico. El merge a la rama principal se realizará **únicamente con TODO el equipo presente** para validar y aprobar los cambios finales en conjunto.

### Gestión y Seguimiento

- **Issues:** Es obligatorio revisar la sección de **Issues** en el repositorio antes de iniciar cualquier tarea para asegurar que el trabajo esté alineado con las metas.
- **Lineamientos:** Consultar los **Lineamientos de Contribuidores** para asegurar el estándar de calidad y estilo del código.
